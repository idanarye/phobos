std.idioms

Reusable implementations of common idioms and design patterns.

This module implements boilerplate code and design patterns that assist in the implementation of the code's architecture.

The Properties idiom assist in defining property member fields - variables access by a pair of @property accessor methods instead of being used directly.
Properties Template mixin used to easily declare property member fields.
Asserting UDA that makes a property setter verify it's input using D's contracts mechanism(assert).
Enforcing UDA that makes a property setter verify it's input using D's exception handling mechanism.

The Singleton idiom turns classes to singletons. There is a version for each one of the three types of globality that D offers:
ThreadLocalSingleton A singleton that is local to the thread. Corresponds to D's static.
SharedSingleton A thread-safe, process-global singleton. Corresponds to D's static shared.
_GSharedSingleton A process-global singleton, providing thread-safe initialization without forcing thread-safe methods. Corresponds to D's __gshared.

class PropertyException: object.Exception;
Exception thrown in case a property is assigned a value not approved by the enforcement condition of an Enforcing UDA.

class SingletonException: object.Exception;
Exception thrown in case a user tries to access a singleton instance for a singleton that was not initialized yet and that has no default constructor.

struct Asserting;
When used as UDA inside Properties, adds an assertion to verify that the value for the property is correct.

Unlike Enforcing, Asserting will be disabled when compiled with the -release flag.

string assertion;
An assertion string that represents the assert condition. The string must use symbol name a as the parameter.

string message;
If not null, the message the assert will throw when the assertion fails.

struct Enforcing;
When used as UDA inside Properties, adds a check to verify that the value for the property is correct, and throw PropertyException if it isn't.

Unlike Asserting, Enforcing will be enabled when compiled with the -release flag.

string enforcement;
An enforcement string that represents the checked condition. The string must use symbol name a as the parameter.

string message;
If not null, the message for the PropertyException.

template Properties(string declarationCode)
Generate property getter and setter accessors for member fields.

The member fields must be supplied as a string with member field declarations. Token string is preferred.

Use Asserting and Enforcing UDAs before properties to verify the values assigned to them.

Parameters:
declarationCode the code to declare the member fields that will be turned into properties.

Examples:
class Foo{
    mixin Properties!q{
        @Asserting(`a <= y`) int x;
        @Enforcing(`a != 0`) double y = 1;
        @Asserting(`a !is this`) typeof(this) z;
    };
}

//typeof(this) works properly
static assert(is(Foo == typeof(Foo.z)));

Foo foo = new Foo();

//Defaults are working
assert(foo.y == 1);

//Assignments that doesn't break the conditions:
foo.x = 1;
assert(foo.x == 1);

foo.y = 4.2;
assert(foo.y == 4.2);

//Assignments that break the conditions throws appropriate errors/exceptions:
assertThrown!AssertError(foo.x = 5);
assertThrown!PropertyException(foo.y = 0);

//Assignments that break the conditions do not change the values:
assert(foo.x == 1);
assert(foo.y == 4.2);

//Conditons operate in the scope of the struct/object:
assertThrown!AssertError(foo.z = foo);

//Works with function local variables:
mixin Properties!q{
    @Asserting("a < 4") int x;
};
assertThrown!AssertError(x = 5);

template _GSharedSingleton()
Turns the class into a singleon, using David Simcha's and Alexander Terekhov's low lock singleton implementation approach.

This version of the Singleton idiom only synchronizes the initialization of the singleton. Methods of the instance are not synchronized automatically by _GSharedSingleton.

This template mixin does not create a private constructor! A private constructor must be declared separately, otherwise the class could be created from outside.

Examples:
//Singleton with default constructor:
static class Foo
{
    mixin _GSharedSingleton;

    private this(){}
}

Foo foo = Foo.instance;

//We initialized it here, so it's initialized in all threads:
auto checkIfFooIsInitializedInAnotherThread = task!(Foo.hasInstance)();
checkIfFooIsInitializedInAnotherThread.executeInNewThread();
assert(checkIfFooIsInitializedInAnotherThread.spinForce());

//Different threads - same instances:
auto getFooFromAnotherThread = task!(Foo.instance)();
getFooFromAnotherThread.executeInNewThread();
assert(foo == getFooFromAnotherThread.spinForce());

//Singleton with no default constructor needs to be initialized:
static class Bar
{
    mixin _GSharedSingleton;

    private int x;

    private this(int x)
    {
        this.x = x;
    }

    private static void init(int x)
    {
        singleton_tryInitInstance(new Bar(x));
    }

}

assertThrown!SingletonException(Bar.instance);

Bar.init(1);
assert(Bar.instance.x == 1);

//Only first initialization works:
Bar.init(2);
assert(Bar.instance.x == 1);

//Even if you try to initialize again from another thread:
auto getBarFromAnotherThread = task!({
        Bar.init(3);
        return Bar.instance;
        })();
getBarFromAnotherThread.executeInNewThread();
assert(Bar.instance == getBarFromAnotherThread.spinForce());
assert(getBarFromAnotherThread.spinForce().x == 1);

template ThreadLocalSingleton()
Turns the class into a thread local singleon.

This template mixin does not create a private constructor! A private constructor must be declared separately, otherwise the class could be created from outside.

Examples:
//Singleton with default constructor:
static class Foo
{
    mixin ThreadLocalSingleton;

    private this(){}
}

Foo foo = Foo.instance;

//Even though we initialized it here, it is still uninitialized in other threads:
auto checkIfFooIsInitializedInAnotherThread = task!(Foo.hasInstance)();
checkIfFooIsInitializedInAnotherThread.executeInNewThread();
assert(!checkIfFooIsInitializedInAnotherThread.spinForce());

//Different threads - different instances:
auto getFooFromAnotherThread = task!(Foo.instance)();
getFooFromAnotherThread.executeInNewThread();
assert(foo != getFooFromAnotherThread.spinForce());

//Singleton with no default constructor needs to be initialized:
static class Bar
{
    mixin ThreadLocalSingleton;

    private int x;

    private this(int x)
    {
        this.x = x;
    }

    private static void init(int x)
    {
        singleton_tryInitInstance(new Bar(x));
    }

}

assertThrown!SingletonException(Bar.instance);

Bar.init(1);
assert(Bar.instance.x == 1);

//Only first initialization in each thread works:
Bar.init(2);
assert(Bar.instance.x == 1);

//But you can still initialize again from another thread:
auto getBarFromAnotherThread = task!({
        Bar.init(3);
        return Bar.instance;
        })();
getBarFromAnotherThread.executeInNewThread();
assert(Bar.instance != getBarFromAnotherThread.spinForce());
assert(getBarFromAnotherThread.spinForce().x == 3);

template SharedSingleton()
Turns the class into a singleon, using David Simcha's and Alexander Terekhov's low lock singleton implementation approach.

This version of the Singleton idiom creates a shared instance of the singleton object, which means that non-shared instance methods will not be accessible.

This template mixin does not create a private constructor! A private constructor must be declared separately, otherwise the class could be created from outside.

Examples:
//Singleton with default constructor:
static shared class Foo
{
    mixin SharedSingleton;

    private this(){}
}

shared(Foo) foo = Foo.instance;

//We initialized it here, so it's initialized in all threads:
auto checkIfFooIsInitializedInAnotherThread = task!(Foo.hasInstance)();
checkIfFooIsInitializedInAnotherThread.executeInNewThread();
assert(checkIfFooIsInitializedInAnotherThread.spinForce());

//Different threads - same instances:
auto getFooFromAnotherThread = task!(Foo.instance)();
getFooFromAnotherThread.executeInNewThread();
assert(foo == getFooFromAnotherThread.spinForce());

//Singleton with no default constructor needs to be initialized:
static shared class Bar
{
    mixin SharedSingleton;

    private int x;

    private this(int x)
    {
        this.x = x;
    }

    private static void init(int x)
    {
        singleton_tryInitInstance(new shared(Bar)(x));
    }

}

assertThrown!SingletonException(Bar.instance);

Bar.init(1);
assert(Bar.instance.x == 1);

//Only first initialization works:
Bar.init(2);
assert(Bar.instance.x == 1);

//Even if you try to initialize again from another thread:

        Bar.init(3);
        return Bar.instance;
        })();
getBarFromAnotherThread.executeInNewThread();
assert(Bar.instance == getBarFromAnotherThread.spinForce());
assert(getBarFromAnotherThread.spinForce().x == 1);
